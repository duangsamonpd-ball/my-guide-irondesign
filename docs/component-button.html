Button

Triggers an action or event — the primary way users move through the product. Built entirely from design tokens, so colour, size, weight, radius and focus all stay in sync with Figma.

6 variants 3 sizes 4 states Token-driven
Preview
Variants
Six semantic variants. Primary for the one main action per view; the rest support it. Outline / Ghost / On-dark use blue text that turns pink on hover.

Four states from Figma — default · hover · focus · disabled. Focus shows a 3px coloured halo matching the button (pink · blue · white @25%); ghost focus uses a blue border. (Loading is a UI extra.)

Code
Copy-and-own HTML — styling comes from the design-system tokens. No framework required.
<!-- variants · size = btn--lg / btn--md / btn--sm -->
<button class="btn btn--md btn--primary">Primary</button>
<button class="btn btn--md btn--secondary">Secondary</button>
<button class="btn btn--md btn--tertiary">Tertiary</button>
<button class="btn btn--md btn--outline">Outline</button>
<button class="btn btn--md btn--ondark">On-dark</button>
<button class="btn btn--md btn--ghost">Ghost</button>

<!-- disabled -->
<button class="btn btn--md btn--primary" disabled>Disabled</button>

<!-- NuGet download CTA -->
<button class="btn-nuget">
  <svg class="nuget-ico"><!-- icon --></svg>
  <span class="nuget-txt">
    <span class="nuget-title">Free NuGet Download</span>
    <span class="nuget-sub">Total download 19,002,577</span>
  </span>
</button>

Guidelines
✓ Do
  • One Primary button per view — the single main action.
  • Pair with Outline / Ghost for secondary actions.
  • Keep labels short, verb-first ("Download", "Start trial").
✗ Don't
  • Don't stack two Primary buttons — they compete.
  • Don't hard-code colours; always use the button tokens.
  • Small (36px) is for desktop / compact & table actions only — use Default (48px) or larger for primary mobile tap targets.
