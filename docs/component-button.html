Button

Triggers an action or event — the primary way users move through the product. Built entirely from design tokens, so colour, size, weight, radius and focus all stay in sync with Figma.

Code
Copy-and-own — plain HTML + CSS variables. No framework required.
<!-- markup -->
<button class="btn btn--lg btn--primary">Download Free Trial</button>

/* styling — all values are tokens */
.btn {
  display: inline-flex; align-items: center; gap: var(--space-xs);
  border-radius: var(--rounded-cta);
}
.btn--lg {
  height: var(--size-btn-lg);          /* 54px */
  padding: 0 26px;
  font-size: var(--font-size-btn-lg);   /* 16px */
  font-weight: var(--fw-btn-lg);        /* 700  */
}
.btn--primary {
  background: var(--color-button-primary);
  color: var(--color-primary-on);
}
.btn--primary:hover  { background: var(--color-button-primary-hover); }
.btn--primary:focus-visible { box-shadow: var(--shadow-focus-magenta); }
Guidelines
✓ Do
  • One Primary button per view — the single main action.
  • Pair with Outline / Ghost for secondary actions.
  • Keep labels short, verb-first ("Download", "Start trial").
✗ Don't
  • Don't stack two Primary buttons — they compete.
  • Don't hard-code colours; always use the button tokens.
  • Don't shrink the tap target below ~40px tall on mobile.
