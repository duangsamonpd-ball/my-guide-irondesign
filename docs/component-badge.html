Badge

A small, non-interactive label that conveys status, category or count. Five semantic colours map straight to the success / warning / danger / info / neutral tokens.

Subtle vs Solid
Subtle (tinted background) is the default for in-page status. Solid is for higher emphasis — use sparingly.

Code
<!-- colours: success / warning / danger / info / neutral -->
<span class="badge badge--success"><span class="bdot"></span>Active</span>
<span class="badge badge--warning">Pending</span>
<span class="badge badge--danger">Failed</span>

<!-- solid + squared -->
<span class="badge solid badge--info">Beta</span>
<span class="badge badge--neutral sq">Draft</span>
.badge {
  font-size: var(--font-size-badge); font-weight: var(--fw-badge);
  border-radius: var(--rounded-full); padding: 5px 10px;
}
.badge--success {        /* subtle */
  background: var(--color-success-subtle);
  color: var(--color-success);
}
.badge.solid.badge--success { background: var(--color-success-solid); color: #fff; }
Guidelines
✓ Do
PaidOverdueArchived
  • Match colour to meaning (green = good, red = error).
  • Keep labels to 1–2 words.
  • Use subtle by default; reserve solid for emphasis.
✗ Don't
Click here to retry now
  • Don't make badges clickable — that's a Button or link.
  • Don't write sentences inside a badge.
  • Don't pick colours for looks; pick by intent token.
