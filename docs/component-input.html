Input

A single-line text field for collecting short user input — email, name, search. All states map to tokens, so focus, error and disabled stay on-brand. The fields below are live.

Guidelines
✓ Do
  • Always pair the input with a visible label.
  • Use the hint for format help, the error for what went wrong.
  • Show the error state only after the user leaves the field.
✗ Don't
  • Don't use placeholder as the only label — it vanishes on type.
  • Don't hard-code the focus colour; use the token.
  • Don't make fields shorter than the 44px tap target.
