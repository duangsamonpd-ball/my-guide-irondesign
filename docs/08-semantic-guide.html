🗺️ Semantic Color Guide for Developers

How to use the design system's color tokens correctly in code — with Tailwind CSS classes, CSS custom properties, and real examples.

What is a semantic token?
❌ Raw value — avoid this
/* Hardcoded hex — breaks when brand changes */
background-color: #E01A59;
border-color: #BA174B;

If the brand colour changes, you'd need to hunt down every `#E01A59` across the whole codebase.

✅ Semantic token — use this
/* Named role — update once, works everywhere */
background-color: var(--color-primary);
border-color: var(--color-primary-hover);

The token name describes the purpose, not the value. Change the token once in tokens.css and everything updates.

// ✅ Typed Button component — semantic tokens via Tailwind
const variantMap = {
  primary:   'bg-primary-500 hover:bg-primary-600 focus:shadow-focus-magenta text-white',
  secondary: 'border-2 border-secondary-500 text-secondary-500'
           + ' hover:bg-secondary-500 hover:text-white focus:shadow-focus-blue',
  danger:    'bg-danger-500 hover:bg-danger-600 focus:shadow-focus-red text-white',
};

export function Button({ variant = 'primary', disabled, children }) {
  return (
    <button
      disabled={disabled}
      className={`
        ${variantMap[variant]}
        font-bold rounded-cta px-xl h-btn-primary
        transition-colors focus:outline-none
        ${disabled ? 'opacity-50 cursor-not-allowed' : ''}
      `}
    >
      {children}
    </button>
  );
}

// Usage

Rules — Do's & Don'ts
✅ DO — Use semantic tokens
color: var(--color-text-body); bg-primary-500 hover:bg-primary-600 border-color: var(--color-border);

Tokens describe purpose. When the brand updates, you change one file, not hundreds.

❌ DON'T — Hardcode raw values
color: #404040; bg-[#E01A59] hover:bg-[#BA174B] border-color: #E2E8F0;

Raw hex codes are invisible to the design system. They break silently when the palette changes.

✅ DO — Match token to purpose
/* Delete button → danger */ bg-danger-500 hover:bg-danger-600 /* Success toast → success */ bg-success-100 text-success-700

Pick the token whose name matches what the UI is communicating to the user.

❌ DON'T — Swap roles for aesthetics
/* Error message in warning colour */ bg-warning-100 text-warning-700 /* Primary CTA in secondary colour */ bg-secondary-500

If you use warning colour for an error, screen readers and colour-blind users get the wrong signal.

✅ DO — Use scale for one-offs
/* Subtle tint on a feature section */ bg-iron-pink-50 /* Chart series colour */ fill: var(--iron-purple-400);

For decorative or data-vis uses where no semantic role fits, raw scale tokens (iron-*) are fine.

❌ DON'T — Invent new colours
/* Not in the design system */ background: #FF5733; color: #3A86FF;

Every colour must come from the token file. New colours need a design review and a token entry first.
